## 目标

将所有会话/消息/面试数据从 MySQL 迁移到 Mongo，与 Java 参考项目对齐。MySQL 只保留用户认证、AI/Agent 配置、文件资产。迁移后删除多余的 MySQL 仓储文件和 GORM tag。

主键策略：会话类用 SessionID 做 Mongo `_id`；记录类（InterviewRecord）用 ObjectID `_id` + sessionId 索引；消息类（已迁）保持 ObjectID。

## 一、模型改造（models/）

### `models/agent.go`
- `AgentConversation`：去掉 `ID uint`、`gorm` tag、`TableName()`；`SessionID` 改为 `bson:"_id"` + `json:"session_id"`；其余字段加 `bson` tag。无 `MongoID`（用 SessionID 做 _id）。
- `AgentMessage`：去掉 `gorm` tag、`TableName()`、`ID uint`（保留 `MongoID` 作 _id）。已是双栖，清理 gorm tag 即可。
- `AgentProperties`/`AgentFileAsset`：不动（留 MySQL）。

### `models/ai.go`
- `AiConversation`：同 AgentConversation 改造，SessionID 做 `_id`，去 gorm tag/TableName/ID。
- `AiMessage`：同 AgentMessage，去 gorm tag/TableName/ID。
- `AiProperties`：不动（留 MySQL）。

### `models/interview.go`
- `InterviewSession`：SessionID 做 `_id`，去 gorm tag/TableName/ID，加 bson tag。
- `InterviewRecord`：用 ObjectID 做 `_id`（加 `MongoID primitive.ObjectID`），SessionID 加 bson tag + 索引，去 gorm tag/TableName/ID。注意：InterviewRecord 一场面试可能多条（每题一条），不能用 SessionID 做 _id。
- `InterviewQuestion`：同 InterviewRecord，ObjectID _id + SessionID 索引，去 gorm tag/TableName/ID。

## 六、interview service 的 ctx 问题

interview_service.go 当前所有方法不带 ctx，全部同步调用。两种处理方式：
- **方案 A（推荐）**：方法内创建 `ctx, cancel := context.WithTimeout(context.Background(), 5*time.Second)`，和 ai_service.go 的 AiMessageService 一致。
- 方案 B：改签名加 ctx 参数——但 handler 层也要改，波及面大。

采用方案 A，最小改动。
